## 🛠️ How to Deploy on GitHub Pages

### Step 1 — Create a GitHub Repository
1. Go to [github.com](https://github.com) and sign in (or create a free account)
2. Click the **+** icon → **New repository**
3. Name it: `leon-richardson-portfolio`
4. Set it to **Public**
5. Click **Create repository**

### Step 2 — Upload Your Files
1. On your new repo page, click **Add file** → **Upload files**
2. Drag and drop both `index.html` and `README.md`
3. Scroll down and click **Commit changes**

### Step 3 — Enable GitHub Pages
1. Go to your repo's **Settings** tab
2. In the left sidebar, click **Pages**
3. Under **Source**, select **Deploy from a branch**
4. Set branch to `main` and folder to `/ (root)`
5. Click **Save**

### Step 4 — View Your Live Site
- Wait ~60 seconds, then visit:
  `https://supaproducer901.github.io/leon-richardson-portfolio/`
- GitHub will show a green banner in Settings → Pages when it's live

## ✏️ Making Updates

To update content on your site:
1. Go to your repository on GitHub
2. Click `index.html`
3. Click the **pencil icon** (Edit)
4. Make your changes
5. Click **Commit changes**

Your live site will update automatically within ~30 seconds.
